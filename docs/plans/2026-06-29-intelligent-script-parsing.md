# 智能文稿解析方案

## 功能概述

为 NarraForge 添加一键智能解析能力，用户粘贴原始文本后，系统自动完成：
- 章节拆分
- 角色识别
- 角色声音特质分析 + 音色推荐
- 台词自动分配给对应角色

目标：将"粘贴文本 → 可以开始配音"的时间从几分钟缩短到 10 秒钟。

---

## 架构设计

### 三层渐进式解析

```
用户粘贴文本
    ↓
[ 第一层：纯正则解析 ] ← 快，零配置，覆盖 80% 场景
    ├─ 拆分章节
    ├─ 识别角色列表
    └─ 粗分配台词到角色
    ↓
[ 第二层：LLM 增强解析 ] ← 可选，用户点击后才运行
    ├─ 优化角色-台词匹配
    ├─ 分析声音特质
    └─ 推荐 TTS 音色参数
    ↓
[ 第三层：用户手动修正 ] ← 永远兜底，AI 永远只是建议
    ├─ 就地编辑角色
    ├─ 批量分配
    └─ 合并/拆分段落
```

---

## 第一层：正则解析规则集

### 1. 章节拆分规则

| 规则 | 正则 | 匹配示例 |
|------|------|---------|
| 中文章节号 | `^第[0-9一二三四五六七八九十百千]+[章节集]\s*(.*)$` | `第1章 夜路`、`第三集 启程` |
| Markdown 标题 | `^#{1,3}\s+(.*)$` | `# 第一章`、`## 第二节` |
| 场景标题 | `^(INT\.|EXT\.|【|△|▲).*$` | `INT. FOREST - NIGHT`、`【第一幕】` |

### 2. 角色识别规则

#### 规则 A：行首冒号格式（最常见）
```regex
^([^：\n\r]{1,12})：
```
匹配：`小明：我们得快点走了！` → 角色：小明

#### 规则 B：带动词的冒号格式
```regex
^([^：\n\r]{1,10})(说|道|答道|问道|怒道|笑道)[:：]
```
匹配：`小明说：我们得快点走了！` → 角色：小明

#### 规则 C：引号 + 后置说话人
```regex
[」"』]\s*([^，。！？\s]{1,8})\s*(说|道)
```
匹配：`"我们得快点走了！"小明焦急地说。` → 角色：小明

#### 规则 D：全大写英文角色名
```regex
^[A-Z\s()]{2,30}$
```
匹配（下一行就是台词）：
```
JOHN
We need to move faster.
```

#### 规则 E：频次过滤（去噪关键）
```
候选角色名出现 ≥ 2 次 → 保留
只出现 1 次 → 标记为"可能"或丢弃
```

### 3. 台词分配规则

匹配到角色名的行，直接分配给对应角色。
未匹配的行 → 默认标记为"旁白"。

## 第二层：LLM 增强解析

### 触发时机
用户在 Library 页面点击「✨ 智能分析角色」按钮才运行，不强制。

### LLM Prompt 设计

```python
SYSTEM_PROMPT = """
你是一个专业的配音导演。根据角色的台词，分析这个角色的声音特质。

输入：角色名 + 该角色的台词样例
输出：JSON 格式，包含以下字段：
{
  "gender": "male" | "female" | "unknown",
  "age_group": "child" | "young" | "middle" | "old",
  "personality": string array, 最多 3 个关键词,
  "speech_style": "fast" | "slow" | "normal",
  "recommended_voice": {
    "engine": "edge_tts" | "mimo" | "cosyvoice",
    "voice_hint": string, 音色描述关键词,
    "speed_adjust": number, -20 到 +20 的整数,
    "pitch_adjust": number, -20 到 +20 的整数
  },
  "confidence": number, 0-1 之间
}

只输出合法的 JSON，不要其他解释或 markdown 标记。
"""
```

### 关键台词挑选规则

每个角色只送 3-5 句代表性台词给 LLM：
1. **长度过滤**：只保留 > 15 个字的句子
2. **情绪关键词优先**：台词附近有「怒道、笑道、平静地说、焦急地」等优先
3. **去掉疑问句和纯感叹**：不选「什么？」「天啊！」这类看不出性格的句子

### 成本估算
假设 3 个角色 × 每个角色 500 token = 1500 token/次
MiMo API 成本约 ¥0.0015/次，几乎可以忽略不计。

## API 设计

### POST /api/text-analysis/split

纯正则拆分，零配置，速度快。

**Request:**
```json
{
  "text": "原始文本内容",
  "mode": "auto" | "script" | "article"
}
```

**Response:**
```json
{
  "method": "regex",
  "chapters": [
    {
      "title": "第1章 夜路",
      "segments": [
        {
          "text": "天色渐暗，远处传来狼嚎。",
          "role": null,
          "role_confidence": 1.0
        },
        {
          "text": "我们得快点走了！",
          "role": "小明",
          "role_confidence": 0.95
        }
      ]
    }
  ],
  "detected_roles": [
    { "name": "小明", "occurrences": 8, "confidence": 0.95 },
    { "name": "小红", "occurrences": 6, "confidence": 0.95 }
  ]
}
```

### POST /api/text-analysis/analyze-roles

LLM 分析角色声音特质，需要用户已配置 LLM API Key。

**Request:**
```json
{
  "text": "原始文本内容",
  "detected_roles": ["小明", "小红"]
}
```

**Response:**
```json
{
  "method": "llm",
  "role_analysis": [
    {
      "name": "小明",
      "gender": "male",
      "age_group": "young",
      "personality": ["急躁", "果断"],
      "speech_style": "fast",
      "recommended_voice": {
        "engine": "edge_tts",
        "voice_hint": "年轻男声",
        "speed_adjust": 10,
        "pitch_adjust": 0
      },
      "confidence": 0.9
    }
  ]
}
```

## 设计原则

### 核心原则

1. **正则永远兜底** — 不启用 LLM 也能 100% 工作
2. **智能是选项，不是必须** — 所有 AI 功能都需要用户主动点击触发
3. **宁可不推荐，也不要乱推荐** — 置信度低就明说，不要假装很懂
4. **用户永远有最终控制权** — AI 只是建议，用户随时可以改

### 体验底线

- 就算有一半推荐得不太对，用户改 3 个角色的音色也只用 30 秒
- 还是比从零开始选快得多

---

## 风险评估

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| 正则识别不准 | 中 | 低 | 标记置信度，用户手动改就行 |
| LLM 输出格式错误 | 低 | 低 | 加 JSON 解析容错，失败就提示重试 |
| 用户没配置 LLM API Key | 高 | 低 | 检测到未配置就灰掉按钮 + 提示去配置 |
| 文本太长 LLM 超 token | 中 | 低 | 每个角色只送 3-5 句代表性台词 |

---

## 预期效果

| 文本类型 | 拆分准确率 | 角色识别准确率 | 音色推荐准确率 |
|---------|-----------|--------------|--------------|
| 标准剧本格式 | 95%+ | 95%+ | 85%+ |
| 网文小说格式 | 85% | 80% | 70% |
| 无格式纯文本 | 50% | 30% | 不推荐 |

**用户体验目标：**
- 标准剧本：粘贴 → 一键拆分 → 直接开始配音（几乎不用改）
- 网文小说：粘贴 → 一键拆分 → 改几个角色分配 → 开始配音
- 纯文本：手动拆分 → 但还是比其他工具快得多
